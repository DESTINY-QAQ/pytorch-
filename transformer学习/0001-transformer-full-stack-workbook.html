Lesson 0001 · Excel by Hand

从一个 token 到下一词概率

这份课件逐格解释 Transformer-Full-Stack.xlsx。核心目标不是记住格子地址，而是看懂每一步的输入、参数、输出和矩阵形状。

先给结论：这个工作簿演示的是一次 forward pass，没有 loss、backpropagation 或 optimizer。训练真正学到的是 Embedding、W_Q/W_K/W_V、FFN、输出层等参数；Q/K/V 则是当前输入乘上这些参数后临时得到的激活。

表格采用一个很关键的约定：列对应 token，行对应特征维度。因此输入写作 d_model × sequence_length，注意力分数写作 KᵀQ。在常见 PyTorch 教材里 token 常放在行上，所以你更多看到 QKᵀ；本质相同。

2. Input Embedding 与 Positional Encoding

源序列有 6 个 token，词表大小 16，首先变成 one-hot 矩阵。一个 5×16 的 Embedding 参数矩阵乘以 16×6 one-hot，得到 5×6 的词向量矩阵：

X_embed = E · OneHot
形状：(5×16)(16×6) = 5×6

随后加上同形状的 sinusoidal Positional Encoding：X = X_embed + PE。Embedding 告诉模型“这是什么词”，位置编码告诉模型“它在第几个位置”。没有位置编码，Self-Attention 本身不能区分词序。

3. Encoder Self-Attention：Q、K、V 从哪里来

本表的 Encoder 使用 2 个 head，每个 head 的 d_k=d_v=3。输入维度是 5，因此三个投影并不要求方阵：

Q = W_Q X　　K = W_K X　　V = W_V X
每个堆叠结果：(6×5)(5×6) = 6×6
拆成 2 个 head 后：每个 Qᵢ、Kᵢ、Vᵢ 都是 3×6

Q 表示当前位置正在寻找什么，K 表示每个位置可被怎样匹配，V 是匹配后真正汇总的信息。它们不是训练结束后固定的三张表；固定的是投影参数，Q/K/V 会随输入改变。

Scaled dot-product attention

Scores = KᵀQ / √d_k
形状：(6×3)(3×6) = 6×6
Weights = softmax(Scores)
Head = V · Weights　→　(3×6)(6×6)=3×6

为什么除以 √d_k？若 Q、K 各维近似独立且方差为 1，点积是 d_k 项之和，方差约为 d_k，标准差约为 √d_k。除掉它后，logit 的典型尺度较稳定，softmax 不容易过早饱和。它不是消除所有维度影响，而是控制点积的统计尺度。

检查 1：为什么这张表写 KᵀQ，论文常写 QKᵀ？
因为本表把 token 放在列上，论文常把 token 放在行上。交换数据布局后，矩阵表达自然转置；每个 query 与所有 key 做点积这一语义没有变化。

Multi-Head 合并

两个 3×6 head 纵向拼接为 6×6，再经 5×6 的 W_O 投影回 5×6，使它能和原输入做 residual addition。

4. Add & Norm 与 Encoder FFN

注意力输出先与进入该子层前的输入相加，再做规范化。Residual connection 保留原信息并给梯度提供短路径；LayerNorm 控制每个 token 的特征尺度。

H₁ = Norm(X + MultiHeadAttention(X))
H₂ = Norm(H₁ + FFN(H₁))

本表 FFN 把特征从 5 扩到 12，再压回 5：5 → 12 → 5。它对每个 token 独立使用同一组参数；注意力负责 token 之间交流，FFN 负责每个 token 内部的非线性变换。

5. Decoder 的三块功能

子层Q 来自哪里K/V 来自哪里作用
Masked Self-Attention已给出的目标前缀已给出的目标前缀整理“已经生成了什么”，但不能偷看未来
Cross-AttentionDecoder 当前状态Encoder 的源句表示针对当前要生成的位置，从源句取相关信息
FFN每个目标位置独立处理非线性提炼每个位置的特征

Causal mask 为什么能阻止偷看

训练 Decoder 时，整句目标 token 可以并行放进矩阵，但位置 t 只能利用不晚于 t 的位置。表中用 -99 近似负无穷，加到禁止的 score 上；softmax 后对应概率几乎为 0。

softmax([可见分数, -99]) ≈ [正常概率, 0]

注意本表分数矩阵的列是 query、行是 key，所以看起来被遮住的是下三角；换成常见“query 在行”的布局后，会显示为上三角 mask。

Cross-Attention 的形状

目标序列预留 8 个位置，源序列有 6 个位置。对每个 head，Decoder query 是 3×8，Encoder key/value 是 3×6：

K_encoderᵀ Q_decoder → (6×3)(3×8)=6×8
V_encoder · Weights → (3×6)(6×8)=3×8

这张 6×8 权重表的每一列回答：“当前目标位置应该看源句的哪几个词？”这就是 Encoder 和 Decoder 联合起来的核心接口。

6. Linear、softmax 与下一词

Decoder 最终状态为 6×8。输出矩阵把每个 6 维状态映射为 24 个词的 logits：

Logits = W_vocab H
(24×6)(6×8)=24×8
P(token | context)=softmax(Logits)

工作簿通过 COUNTA(...)-1 找到最后一个已填写目标位置 5，也就是 play，再取该列 24 个概率。最高几项如下：

候选词概率
fill62.1668%
light11.6669%
smile9.7670%
the5.6851%
play4.4487%

如果使用 greedy decoding，就选择 fill；若使用 sampling，则按整个分布随机抽样。选出的 token 会追加到目标序列，再重新运行 Decoder，循环生成。

7. 它与 GPT 的关系

原始 Transformer

Encoder 读完整源输入；Decoder 通过 causal self-attention 读目标前缀，并通过 cross-attention 查询 Encoder。适合翻译等 sequence-to-sequence 任务。

GPT

本质是 Decoder-only：保留 causal self-attention 和 FFN，不使用独立 Encoder，也没有 Encoder–Decoder cross-attention。提示词和已生成 token 都在同一条序列里。

因此“GPT 预测下一个词”的直觉是对的，但更准确的说法是：给定已有 token，模型通过多层 masked self-attention 计算最后位置的隐藏状态，再映射到整个词表的下一个 token 概率。训练通过预测误差更新所有参数，而不是训练一次后只存下一词。

8. 这份工作簿的教学简化与异常

  • 并未展示训练：没有 target loss、反向传播和参数更新，权重是预填的演示值。
  • SOFTMAX 数值稳定性：命名公式直接使用 EXP(x)/SUM(EXP(x))；工程实现通常先减 MAX(x)，概率不变但更不易溢出。
  • LayerNorm 是简化版：NORMALIZE 使用 AVERAGE、样本标准差 STDEV 和 epsilon 0.001，不完全等同于现代框架中的 LayerNorm 实现。
  • FFN 的第二次 ReLU：本表在第二个 linear 后还出现 ReLU；标准 Transformer 通常只在两层 linear 之间激活。
  • 公式异常：causal mask 的命名区域公式中存在字面量 #REF!。若 Excel 中出现错误，应优先修复该引用。
  • 兼容性：MAKEARRAY、LAMBDA、动态数组等需要较新的 Microsoft Excel；第三方渲染器可能显示 #VALUE!，不必立刻判定原文件已损坏。

9. 现在最值得掌握的 20%

  1. 参数是 W，Q/K/V 是由当前输入算出的中间值。
  2. Attention 的本质是“匹配 Q 和 K，再按权重汇总 V”。
  3. √d_k 控制 logit 尺度；mask 控制信息可见性。
  4. Encoder 做双向理解，Decoder 做受约束生成，cross-attention 把二者接起来。
  5. GPT 删除了独立 Encoder 和 cross-attention，但“最后状态 → logits → softmax → 下一 token”的主线不变。
检查 2：训练真正更新的是 Q/K/V，还是 W_Q/W_K/W_V？
更新的是 W_Q/W_K/W_V 等模型参数。Q/K/V 是每次 forward 时由当前输入和这些参数相乘得到的激活；换一句输入，它们就会改变。
