Transformer 做的核心事情：让每个 token 根据上下文更新自己的“理解”，最后预测接下来最可能出现的 token。

5×6 读作“5 行、6 列”。本 Excel 把 一个 token 放在一列，所以 6 列代表 6 个 token；每列有 5 个数字，所以每个 token 暂时用 5 维向量表示。

每张卡有 5 个属性数字。6 张卡并排，就是 5×6 矩阵。矩阵乘法可以理解为：按照一套规则重新组合每张卡上的属性。

0. 先听完整故事，再看大表

假设任务是把中文“小 狗 和 小 猫 玩”翻译成英文。可以把整个 Transformer 想成两个人合作：

🧑‍🏫 Encoder：阅读员

把完整中文句子读一遍，为每个词写一份结合上下文的笔记。它不负责直接输出英文。

✍️ Decoder：写作员

一边看已经写出的英文，一边查阅 Encoder 的中文笔记，逐个写出下一个英文 token。

中文变成数字token → one-hot → Embedding → 加入位置。
Encoder 理解中文每个中文 token 关注句中其他 token，得到带上下文的表示。
Decoder 整理已写内容它只能看已生成的英文，不能偷看未来答案。
Decoder 查询中文笔记通过 Cross-Attention 从 Encoder 输出中寻找当前需要的信息。
产生下一词概率把当前隐藏状态映射到 24 个候选词，使用 softmax 得到概率。
只记这一句Encoder 负责“读懂输入”，Decoder 负责“根据已写内容和输入笔记继续写”。

1. 文字为什么必须先变成数字？

神经网络不能直接对“狗”这个汉字做乘法，所以先为它分配 token ID，再把 ID 变成向量。

Token ID给词表中每个 token 一个编号。例如“狗”可能是 0，“猫”可能是 1。
One-hot用只有一个 1 的长向量指出“我要词表里的哪一个 token”。
Embedding从 Embedding 参数表中取出该 token 对应的一列数字。
Positional Encoding给向量加入位置信息，让相同的词出现在不同位置时可以被区分。

从 one-hot 到向量：完整小例子

假设词表只有 3 个词，Embedding 表如下。每一列对应一个词：

维度狗猫玩
第 1 维274
第 2 维138

“猫”的 one-hot 是 [0,1,0]ᵀ。矩阵乘法：

[[2,7,4], [1,3,8]] × [0,1,0]ᵀ
= [7,3]ᵀ

你可以把它理解成“第 2 个位置为 1，所以选出第 2 列”。原表用 Z41=MMULT(C41:R45,Z21:AE36) 同时选出 6 个 token 的向量。

为什么还要位置编码？“狗追猫”和“猫追狗”使用相同的三个词，但意义不同。Embedding 只告诉模型词是谁，位置编码告诉模型词在哪里。

2. Q、K、V 到底是什么？

🔎

把 Attention 想成搜索资料

Q（Query）是“我现在想找什么”；K（Key）是“每份资料贴着什么标签”；V（Value）是“资料里面真正的内容”。Q 与 K 越匹配，就越多读取对应的 V。

例如在句子“银行在河边”中理解“银行”时，Query 可能在寻找“这是金融机构还是河岸？”；“河边”的 Key 与地点语境强相关；它的 Value 提供真实上下文信息，帮助更新“银行”的含义。

⚠️
最常见误解：模型训练后不是得到一套固定 Q/K/V。训练更新的是 W_Q/W_K/W_V；每次输入新句子，模型都会重新计算新的 Q/K/V。

人话版本

对输入中的每个 token，同时制作三张不同用途的卡片：提问卡 Q、标签卡 K、内容卡 V。

公式版本

Q=W_QX
K=W_KX
V=W_VX

预览截图中的部分 #VALUE! 是渲染器不完全支持 Excel 动态数组造成的显示问题；原单元格公式仍可用于追踪计算逻辑。

展开查看本表 Q/K/V 的形状

Encoder 输入 X 是 5×6。Wq/Wk/Wv 都把 5 维输入投影成总计 6 维：

W_Q：6×5　X：5×6
Q=(6×5)(5×6)=6×6

K、V 同样为 6×6。随后沿特征维拆成 2 个 head，所以每个 head 的 Q、K、V 都是 3×6。

停下来确认

如果换了一句输入，下面哪一项一定会重新变化？

A. Wq/Wk/Wv　　B. Q/K/V

B。参数 W 通常保持不变，Q/K/V 会随着当前输入重新计算。

3. Attention 怎样从“匹配”变成“信息汇总”？

Attention 分三步：先打分，再把分数变成比例，最后按比例读取内容。

① Q 与 K 点积
得到匹配分数
→
② 除以 √dₖ
控制分数尺度
→
③ Softmax
变成关注比例
→
④ 加权 V
得到上下文信息

完整算例：一个 Query 关注两个 token

假设当前 Query 与两个 Key 的点积分数分别是 1 和 2。先暂时认为它们已经缩放过：

e¹=2.71828　e²=7.38906
总和=2.71828+7.38906=10.10734
权重₁=2.71828/10.10734≈0.269
权重₂=7.38906/10.10734≈0.731

若两个 Value 是 V₁=[2,0]、V₂=[0,4]：

输出 = 0.269×[2,0] + 0.731×[0,4]
= [0.538,0] + [0,2.924]
= [0.538,2.924]

所以 Attention 的输出不是某个原词，而是多个 Value 按相关程度混合后的新向量。

为什么除以 √dₖ？

维度越多，点积通常越容易变大。分数太大时，softmax 会变得极端，几乎只给最大值概率，其他位置接近 0。除以 √dₖ 是为了把分数尺度拉回较稳定的范围。

🎚️

像给音量加一个自动增益控制

维度增加会让点积分数“音量变大”。除以 √dₖ 不是删除信息，而是调节整体音量，避免 softmax 被过大的差距推到饱和状态。

展开查看为什么恰好是 √dₖ

若 Q、K 每一维近似独立、均值为 0、方差为 1，那么点积由 dₖ 项相加，方差约为 dₖ，标准差约为 √dₖ。除以 √dₖ 后，结果的标准差回到约 1。

为什么要 Multi-Head？

只用一个 head，就像只用一种标准阅读句子。多个 head 可以在不同投影空间中同时寻找不同关系：某个 head 可能更关注语法搭配，另一个可能更关注指代或语义相关性。它们最后拼接，再用 W_O 合并。

Attention 人话公式提出问题 Q → 用标签 K 判断该看谁 → softmax 决定各看多少 → 汇总对应内容 V。

4. Encoder 为什么还需要 Add & Norm 和 FFN？

➕

Residual / Add

把子层输出加回原输入。模型学习“在原信息上修改多少”，而不是每层从零重建。

⚖️

Normalization

把数值尺度整理得更稳定，避免某些维度长期过大或过小。

🧠

FFN

每个 token 独立进入同一套小型神经网络，进行非线性加工。

Residual 小例子

原向量 x=[2,-1]，Attention 建议修改 Δ=[0.3,0.8]：

x+Δ=[2.3,-0.2]

原信息 2 和 −1 没有被完全丢弃，只是在其基础上做调整。

Attention 在做什么？

让一个 token 从其他 token 读取信息，解决“词与词之间如何交流”。

FFN 在做什么？

对每个 token 自己的特征做加工，解决“读到信息后如何思考”。

工作簿的教学简化：本表 Normalization 使用 Excel 命名公式近似实现；FFN 第二个 Linear 后还出现 ReLU。标准 Transformer 通常只在两个 Linear 之间使用一次激活。

5. Decoder：为什么输入要右移，还要加 mask？

Decoder 的任务是逐个写词。它写当前词时，只能使用已经写出的内容。如果训练时让它看见未来的正确答案，它就会“作弊”。

右移一格：让输入和答案错开

Decoder 已经看到当前应该学会预测
[S]the
[S] thedog
[S] the dogand
[S] the dog andcat

训练时可以并行计算这些位置，但 causal mask 保证每个位置仍然只能使用左侧历史。

Mask 完整小例子

假设位置 1 的原始 attention 分数是 [1,2,3]，但第 3 项来自未来位置：

原分数：[1,2,3]
mask：[0,0,-99]
相加：[1,2,-96]
softmax≈[0.269,0.731,0]

未来位置原本分数最高，但经过 mask 后几乎完全不可见。

↘️
为什么本表看起来遮住下三角？因为本表把 Query 放在列、Key 放在行。很多教材的布局相反，因此显示为上三角。不要死记三角方向，要检查“某个 Query 是否只能看到自己和过去”。

6. Cross-Attention：写英文时怎样查看中文？

Decoder 先通过 masked self-attention 整理“我已经写了什么”，然后把当前需求变成 Query，去查询 Encoder 为中文句子准备的 Key 和 Value。

Self-Attention

Q、K、V 来自同一侧。Encoder 内部读中文，或 Decoder 内部读已生成英文。

Cross-Attention

Q 来自 Decoder，K/V 来自 Encoder。写作员带着问题查询阅读员的笔记。

生成 dog 时的直觉算例

假设当前 Query 对源词“小 / 狗 / 玩”的权重为 [0.1,0.8,0.1]：

V小=[1,0]　V狗=[0,3]　V玩=[2,1]
Context=0.1V小+0.8V狗+0.1V玩
=[0.1,0]+[0,2.4]+[0.2,0.1]
=[0.3,2.5]

模型主要读取“狗”的 Value，同时保留少量相邻语境。

展开查看本表 Cross-Attention 的形状
Decoder Query：3×8
Encoder Key/Value：3×6
KᵀQ：(6×3)(3×8)=6×8
V·Weights：(3×6)(6×8)=3×8

6×8 表示：对于 8 个目标位置中的每一个，都有一列对 6 个源位置的关注权重。

8. 这份表为什么是“预测”，却不是“训练”？

Forward / 预测计算

使用当前参数，从输入一路算到下一词概率。这正是 Excel 展示的内容。

Training / 训练

还必须知道正确答案，计算 loss，反向传播梯度，再由 optimizer 修改参数。

❌
不要这样说：“给定输入训练后得到 QKV 矩阵，再固定用它预测。”
更准确：训练得到参数 W；每次 forward 用当前输入和 W 重新生成 Q/K/V，再预测下一个 token。

9. 这张 Encoder–Decoder 表与 GPT 有什么关系？

📖

原始 Transformer

Encoder 读取源序列，Decoder 生成目标序列，并用 Cross-Attention 连接两边。

💬

GPT

Decoder-only。提示词与已生成内容放在同一序列中，只使用 causal self-attention。

🖼️

CLIP

分别用图像 Encoder 与文本 Encoder 得到表示，再训练两种表示相互匹配；它不是逐词生成器。

GPT 的生成主线：已有 token → masked self-attention → 最后位置隐藏状态 → 词表 logits → softmax → 下一个 token。

GPT 生成循环

输入：“今天天气”
→ 预测“很”
→ 新输入：“今天天气很”
→ 预测“好”
→ 新输入：“今天天气很好”
→ 继续预测……

每多生成一个 token，都要把它加入上下文，再计算下一步。现代实现会缓存以前的 K/V 来减少重复运算，这叫 KV cache。
