Embedding：编号变坐标

Token ID 只是标签，没有大小或语义。Embedding 是一个可学习的查找表，把每个 ID 映射成浮点向量。

课程主页形状速查
ID: 42
查表 E[42]
向量: [0.12, -0.31, ...]
加入位置信息

最小数学

词表大小为 V、隐藏维度为 C 时，嵌入矩阵 E 的形状是 [V, C]。输入 token IDs 是 [B, T]，查表后得到 [B, T, C]。

x = token_embedding(token_ids) # [B, T, C] x = x + position_embedding(positions) # [B, T, C]

位置嵌入不是装饰：注意力本身不认识顺序。没有位置信息，“我爱你”和“你爱我”只是一组相同 token 的重排。

静态向量与上下文向量

表示同一个词在不同句子中例子
Word2Vec / GloVe始终同一向量。“苹果公司”和“吃苹果”里的“苹果”相同
Transformer hidden state经过注意力后随上下文改变。两个“苹果”会得到不同表示

容易混淆：Transformer 也有一个静态的 token embedding 表，但模型真正用于理解的“上下文 embedding”是经过多层 Transformer 后的隐藏状态。

嵌入怎么学到的？

1
给任务：GPT 型模型根据前文预测下一个 token；BERT 型模型根据上下文预测被遮住的 token。
2
算误差：预测分布和真实 token ID 比较，得到交叉熵损失。
3
反向传播：误差穿过输出层、注意力层，最后更新被用到的 embedding 行。
正确心智模型：embedding 不是“预先查到的语义答案”。训练开始时它是随机表；它因为帮助完成预测任务，才逐渐得到有用的几何结构。
