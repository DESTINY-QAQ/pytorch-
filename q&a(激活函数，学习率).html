🧠 神经网络核心概念问答合集

一、隐藏层节点为什么使用激活函数的不同部分？

1. 激活函数有不同“性格”的区域

  • Sigmoid / Tanh：有饱和区（两端平缓，梯度接近0）和线性区（中间陡峭，梯度较大）。
  • ReLU：有死区（x<0，输出0，梯度0）和激活区（x>0，输出=x，梯度1）。

2. 不同神经元为什么落在不同区域？

  • 初始化阶段：权重和偏置随机初始化，不同神经元的加权和 z = Wx + b 大小不一、正负各异，天然落在激活函数的不同位置。
  • 训练阶段：反向传播时，处于不同区域的神经元拿到的梯度差异极大。饱和区/死区的神经元几乎不更新，线性区/激活区的神经元被积极更新，分工被固化强化。

3. 分工的好处

打破对称性，学习更丰富的特征。 如果所有神经元都使用激活函数的同一部分，网络就退化成一个宽线性模型。正是“有的饱和、有的活跃、有的关闭”的组合状态，让网络能拼出复杂的非线性决策边界。

ReLU 举例

  • 神经元 A：z 为正 → 激活区，输出正数
  • 神经元 B：z 为负 → 死区，输出0
  • 神经元 C：z 为正且大 → 激活区，输出大值

三个神经元使用 ReLU 的不同片段，提供多样化信号。

二、激活函数的选择方法

1. 输出层：由任务决定

任务类型激活函数原因
回归（预测房价）无（线性）输出任意实数
二分类（垃圾邮件）Sigmoid输出 (0,1) 概率
多分类（图像识别）Softmax输出总和为1的概率分布
多标签分类Sigmoid（每节点独立）每个标签独立的二分类

2. 隐藏层：解决梯度消失是核心

  • Sigmoid / Tanh：过去常用，但有饱和区导致梯度消失的问题，现代深度网络基本不用。
  • ReLU：max(0, x)，计算快，正区间梯度恒为1，缓解梯度消失。是隐藏层的默认首选。缺点是可能“死亡”（永久输出0）。用 He 初始化，可搭配 BatchNorm。
  • Leaky ReLU / PReLU：负区间给小斜率，防止神经元死亡。ReLU 死神经元多时可以尝试。
  • ELU / SELU：负区间指数函数，输出均值接近0。SELU 可实现自归一化，在深层全连接网络（无 BN）中很强。
  • GELU：BERT、GPT 等 Transformer 模型标配。
  • Swish / SiLU：x * sigmoid(x)，高效网络中表现优异。LLaMA 等使用其变体 SwiGLU。
  • RNN/LSTM/GRU：门控用 Sigmoid，状态更新用 Tanh，依然是标配组合。

3. 决策路线图

  1. 输出层：看任务类型。
  2. 隐藏层：首选 ReLU（He 初始化 + BatchNorm）。
  3. 学不好 / 死神经元多：换 Leaky ReLU 或 ELU。
  4. Transformer 架构：用 GELU 或 SwiGLU。
  5. RNN 架构：Tanh + Sigmoid 组合。
  6. 深层无 BN 全连接：可挑战 SELU。

三、反向传播中学习率的作用

核心关系：梯度提供方向，学习率控制步长

w_new = w_old - η * (∂L/∂w)
  • 梯度 ∂L/∂w：指出损失上升最快的方向（加负号即下降方向）。
  • 学习率 η：缩放梯度，决定一次迈多大步。

下山比喻

  • 太小：步子碎，收敛极慢，可能困在浅坑里。
  • 刚好：稳定快速地接近谷底。
  • 太大：跨过谷底，在两侧震荡，甚至飞出去爆炸（NaN）。

学习率调度

  • 衰减：初期大步快走，后期小步精调。
  • 自适应（如 Adam）：每个参数独立调整学习率，频繁更新的参数步小，反之步大。
总结：学习率统一缩放参数更新的幅度，是训练中最关键的超参之一。

四、局部最优解会过拟合吗？

核心结论：局部最优解不一定导致过拟合，甚至可能是天然防过拟合机制。两者没有必然因果关系。

概念分清

  • 局部最优：优化问题，训练损失降不下去了。
  • 过拟合：泛化问题，训练损失低但测试损失高。

三种情况分析

A. 局部最优可能导致欠拟合
如果陷入很差的局部最优，训练损失本身还很高，连训练集都没学好，谈不上过拟合。

B. 局部最优可能防过拟合
全局最优往往完美拟合了训练数据的噪声。而一个稍高一点的局部最优，反而因“学得不够完美”而忽略了噪声，泛化更好。这就是早停法的原理。

C. 关键在于“平坦”还是“尖锐”

极小值类型地形特征泛化能力过拟合风险
平坦极小值宽阔盆地，参数扰动损失仍低好（学到稳定规律）低
尖锐极小值深井，参数一动损失飙升差（死记噪声细节）高

全局最优可以是尖锐的，局部最优可以是平坦的。平坦的局部最优往往是我们真正想要的解。SGD 的随机性本身就倾向于找到这种平坦解，这也是深度学习泛化能力强的原因之一。
